    Wiki Documentation & Formatting System

    You now have a complete system for creating consistent, professional wiki pages. This index helps you navigate all the resources.

    📚 What You Have

    A comprehensive formatting and templating system with:

    • 📋 Formatting Rules — Comprehensive standards for all content
    • 🎨 Brand Guidelines — Colors, design tokens, visual identity
    • 📄 Page Templates — Ready-to-use structures for 6 different page types
    • ⚡ Quick Reference — One-page syntax cheat sheet
    • 🤖 Copilot Prompts — Ready-to-use prompts for consistent AI generation
    • 📖 Component Library — Detailed guides for each wiki component

    🚀 Quick Start

    For Creating a New Page

    1. Visit: wiki_design/README.md — Master index of all resources
    2. Choose a template from wiki_design/page_templates/
    3. Follow the formatting guide — WIKI_FORMATTING_GUIDE.md
    4. Use the quick reference — wiki_design/QUICK_REFERENCE.md

    For Using with Copilot

    1. Open wiki_design/COPILOT_PROMPTS.md
    2. Choose the prompt template for your page type
    3. Customize with your specific content
    4. Paste into Copilot and let it generate the page

    📋 Pre-Publishing Checklist

    Before publishing any page:

    • YAML front matter present (title, description)
    • Heading hierarchy correct (one #, then ##, ###)
    • All internal links relative (../path/page.md)
    • All code blocks have language hints
    • All images have alt text
    • No hardcoded colors — using CSS variables
    • Admonition types appropriate for content
    • Emphasis (bold/italic) used sparingly
    • Content clear and concise
    • Grammar and spelling correct
    • Tested in light AND dark modes

    See full checklist in wiki_design/QUICK_REFERENCE.md

    🎨 Design Standards Summary

    Essential Rules

    ✅ Always: - Include YAML front matter with title and description - Use one # heading per page - Follow heading hierarchy - Use relative links for internal pages - Include alt text for images - Test in both light and dark modes

    ❌ Never: - Skip YAML front matter - Use multiple # headings - Skip heading levels - Hardcode colors - Hide critical warnings in collapsible blocks

    🤖 Using with Copilot

    Basic Workflow

    1. Decide page type — What kind of page are you creating?
    2. Find prompt template — Open COPILOT_PROMPTS.md
    3. Customize — Replace [placeholders] with your content
    4. Paste — Send to Copilot
    5. Review — Check the output and use the checklist above

    Example Prompt

    Create a new standard wiki page about [Your Topic].

Use the standard_page template from wiki_design/page_templates/standard_page.md

Page structure:
- YAML front matter with title and description
- H1 heading: [Your Topic]
- Introduction explaining the topic
- 2-3 main sections covering [key points]
- Include tip admonitions for best practices
- See Also section with related pages

Follow all standards in WIKI_FORMATTING_GUIDE.md
